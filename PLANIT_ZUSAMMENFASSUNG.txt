PLANIT - ZUSAMMENFASSUNG FÜR NEUEN CHAT
========================================
Datum: 3. April 2026

WAS WIR BAUEN:
--------------
PlanIt ist eine eigenständige Mitarbeiter-App für Schichtplanung.
Komplett separate App, nutzt gleiche Supabase-Datenbank wie GastroHub.
Deployed auf Cloudflare Pages: planit-886.pages.dev

TECHNISCHER STACK:
------------------
- Vanilla HTML/CSS/JS (keine Frameworks)
- Supabase als Backend (gleiche DB wie GastroHub)
- Supabase URL: https://ulazvcrpasnhufzlywxf.supabase.co
- GitHub: github.com/ArtsemKozel/planit
- Cloudflare Pages: planit-886.pages.dev

DATENBANK-TABELLEN:
-------------------
1. employees_planit (id uuid, user_id, name, login_code, password_hash, department,
   is_active, birthdate, vacation_days_per_year, is_apprentice, start_date,
   hours_per_vacation_day, can_do_inventory boolean)

2. shifts (id, user_id, employee_id, shift_date, start_time, end_time, break_minutes,
   notes, is_open, open_note, department, handover_requested, handover_employee_id,
   actual_start_time, actual_end_time, actual_break_minutes)
   → actual_* Felder: tatsächliche Zeiten, werden für Stundenkonto und tip_hours genutzt

3. vacation_requests (id, user_id, employee_id, start_date, end_date, reason, status,
   rejection_reason, reviewed_at, reviewed_by, deducted_days, type, payout_month, pdf_url)

4. availability, shift_swaps, shift_handovers, payroll_documents (wie vorher)

5. approved_hours (id, user_id, employee_id, month, approved_minutes)

6. actual_hours (id, user_id, employee_id, month, actual_minutes, carry_over_minutes)
   → actual_minutes wird nicht mehr manuell eingetragen; carry_over_minutes bleibt

7. open_shift_requests, shift_templates, sick_leaves, employment_phases (wie vorher)

8. tasks (id, user_id, title, created_at, is_archived)
9. task_steps (id, user_id, task_id, title, is_done, position)
10. task_templates, task_template_steps, notes (wie vorher)

11. tip_config (id, user_id, mode, show_to_employees, fixed_results jsonb)
12. tip_departments (id, user_id, department, percentage, fixed_hours_per_month, pool_department)
13. tip_entries (id, user_id, entry_date, amount_card, amount_cash)
    → Unique Constraint: (user_id, entry_date)
    → Alle Tage des Monats werden beim Laden automatisch angelegt (upsert)
14. tip_hours (id, user_id, employee_id, work_date, minutes, start_time, end_time, break_minutes)
    → Unique Constraint: (user_id, employee_id, work_date)
    → Wird automatisch aus shifts befüllt (Sync beim Laden + nach Schicht-Änderungen)
    → start_time, end_time, break_minutes werden mitgespeichert
15. tip_results (id, user_id, employee_id, month, amount_card, amount_cash)

16. planit_suppliers (id, user_id, name, created_at)
17. planit_inventory_items (id, user_id, supplier_id, name, unit, target_amount,
    position, inventory_position, order_position, price_per_unit, group_id, created_at)
    → group_id: Zuordnung zu Warengruppe
18. planit_inventory_entries (id, user_id, item_id, entry_date, actual_amount, created_at)
19. planit_inventory_groups (id, user_id, supplier_id, name, position, created_at)
    → Warengruppen pro Lieferant, klappbar, benennbar
20. planit_inventory_submissions (id, user_id, employee_id, entry_date, status, created_at)
    → RLS aktiviert; Mitarbeiter schreibt, Admin liest

WICHTIGE TECHNISCHE DETAILS:
-----------------------------
- Supabase-Client MUSS "db" heißen
- Modal öffnen: classList.add('active') — NICHT 'open'! (Ausnahme: shift-modal, swap-modal)
- .maybeSingle() statt .single()
- Timezone Bug Fix: new Date(date + 'T12:00:00')
- shift_handovers + open_shift_requests + shift_swaps (beide Richtungen) löschen VOR shifts löschen
- renderWeekGrid ist async
- updateShiftCell() aktualisiert nur eine Zelle (kein Scroll-Reset)
- refreshHoursOverview() aktualisiert nur die deptStundenDivs ([data-stunden-dept])
- Organiser: tasks-archive-list (nicht archive-list — Konflikt mit Urlaub)
- Trinkgeld: saveTrinkgeldResults() nach jedem saveTrinkgeldHours()
- Inventur: inventory_position für Inventur-Reihenfolge, order_position für Bestell-Reihenfolge
- Inventur: Positionen werden beim Laden von loadInventurConfig() initialisiert
- tip_entries Unique Constraint: user_id, entry_date
- tip_hours Unique Constraint: user_id, employee_id, work_date
- Trinkgeld-Sync: actual_break_minutes ?? break_minutes ?? 0 (nicht ||, wegen 0-Wert)
- Stundenkonto-Toggle: <button> statt <div> für Mobile onclick-Kompatibilität
- Schichtplan: korrigierte Schichten (actual_start_time gesetzt) → background #E8D4A0
